The Avatar component is used to represent a user or entity through an image, initials, or icon. It supports different sizes and can be used in various contexts like user profiles, comments, and navigation.

## Usage

```jsx
<Avatar src="https://example.com/avatar.jpg" alt="John Doe" fallback="JD" />
```

## Props

### Avatar

| Prop       | Type                                        | Default    | Description                       |
| ---------- | ------------------------------------------- | ---------- | --------------------------------- |
| `src`      | `string`                                    | -          | URL of the avatar image           |
| `alt`      | `string`                                    | -          | Alternative text for the image    |
| `fallback` | `string \| ReactNode`                       | -          | Fallback content when image fails |
| `size`     | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl'`      | `'md'`     | Size of the avatar                |
| `shape`    | `'circle' \| 'square'`                      | `'circle'` | Shape of the avatar               |
| `status`   | `'online' \| 'offline' \| 'away' \| 'busy'` | -          | Optional status indicator         |

### AvatarGroup

| Prop      | Type                                   | Default    | Description                       |
| --------- | -------------------------------------- | ---------- | --------------------------------- |
| `max`     | `number`                               | -          | Maximum number of avatars to show |
| `size`    | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl'` | `'md'`     | Size of all avatars               |
| `spacing` | `'tight' \| 'normal'`                  | `'normal'` | Space between avatars             |

## Accessibility

- Images have meaningful alt text
- Fallback text is readable by screen readers
- Status indicators have appropriate ARIA labels
- Interactive avatars are keyboard accessible

## Best Practices

1. **Image Handling**

   - Use appropriate image sizes
   - Handle loading states gracefully
   - Provide meaningful fallbacks
   - Consider image optimization

2. **Visual Design**

   - Maintain consistent sizing
   - Use appropriate border radius
   - Consider background contrast
   - Handle long text in fallbacks

3. **Performance**
   - Lazy load images when possible
   - Use responsive images
   - Cache avatar resources
   - Optimize group rendering

## Examples

### Basic Avatar

```jsx
<Avatar src="https://example.com/user.jpg" alt="User Name" size="md" />
```

### With Fallback

```jsx
<Avatar fallback="JD" alt="John Doe" size="lg" />
```

### Avatar Group

```jsx
<AvatarGroup max={3}>
	<Avatar src="https://example.com/user1.jpg" alt="User 1" />
	<Avatar src="https://example.com/user2.jpg" alt="User 2" />
	<Avatar src="https://example.com/user3.jpg" alt="User 3" />
	<Avatar src="https://example.com/user4.jpg" alt="User 4" />
</AvatarGroup>
```

### With Status

```jsx
<Avatar src="https://example.com/user.jpg" alt="Online User" status="online" />
```

### Different Sizes
