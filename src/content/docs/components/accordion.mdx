The Accordion component is used to organize and hide content behind expandable sections, helping to reduce visual clutter while keeping information accessible.

## Usage

```jsx
<Accordion>
	<AccordionItem>
		<AccordionTrigger>Is it accessible?</AccordionTrigger>
		<AccordionContent>Yes. It adheres to the WAI-ARIA design pattern.</AccordionContent>
	</AccordionItem>
</Accordion>
```

## Props

### Accordion

| Prop            | Type                     | Default    | Description                                 |
| --------------- | ------------------------ | ---------- | ------------------------------------------- |
| `type`          | `'single' \| 'multiple'` | `'single'` | Determines if multiple sections can be open |
| `defaultValue`  | `string`                 | -          | The value of the item to expand by default  |
| `value`         | `string`                 | -          | The controlled value of the item to expand  |
| `onValueChange` | `function`               | -          | Event handler called when value changes     |

### AccordionItem

| Prop       | Type      | Default | Description                          |
| ---------- | --------- | ------- | ------------------------------------ |
| `value`    | `string`  | -       | A unique value for the item          |
| `disabled` | `boolean` | `false` | When true, prevents user interaction |

## Accessibility

- Uses appropriate ARIA attributes (`aria-expanded`, `aria-controls`)
- Keyboard navigation support (Space, Enter, Arrow keys)
- Focus management between trigger and content

## Best Practices

1. **Content Organization**

   - Group related content together
   - Use clear and concise headers
   - Keep content length manageable

2. **Interaction Design**

   - Provide clear visual feedback for state changes
   - Maintain consistent animation timing
   - Consider mobile touch targets

3. **Performance**
   - Lazy load content when possible
   - Optimize animations for smooth transitions
   - Consider content reflow on expand/collapse
